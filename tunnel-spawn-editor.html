TUNNEL RUNNER — SPAWN CONFIG

Adjust spawn parameters per sector & level. Export JSON for Claude.
SPAWN DISTRIBUTION
Effective chances after barrier/boost/star/content are subtracted
OBJECT GATING
Which object types are allowed to appear in this sector/level. This is the master switch — if something is OFF here, it won't spawn regardless of other settings.
BASE SPAWN RATIOS
The core recipe. Every time the game places an object, it rolls the dice — these percentages decide what you get. Hazard + Ring + Core should roughly add up to 1.0 (the system normalizes them).
BARRIERS
Barriers are checked BEFORE the regular spawn roll. If a barrier triggers, it replaces that spawn point entirely. Full-ring barriers always come with a ramp so the player can jump over.
SPEED RUN — BARRIER SERIES
In speed run sectors, barriers come in clusters — like a ski slalom. Each cluster is a group of gap barriers with the opening rotating slightly between each one, forcing the player to spiral through.
SPACING & DENSITY
Controls how tightly packed everything is. The tunnel is populated by walking forward from the start and placing one object every [spacing] units. Think of spacing as the distance between fence posts — closer posts = denser obstacle course.
SPECIAL OBJECTS
These have their own independent chances, checked before the main hazard/ring/core roll. They're rarer items that add variety — stars for defense, content for portfolio unlocks, boost pads for spectacle.
HAZARD TYPE MIX
When the dice says "hazard", this decides WHICH hazard. The three types have different shapes and behaviors — mines are small and static, blocks are wide, knots move unpredictably.
SECTOR PHYSICS
Speed and length define the feel of the sector. Fast + short = intense burst. Slow + long = endurance run. Each level replay scales these up automatically (speed ×1.1, length ×1.15).
